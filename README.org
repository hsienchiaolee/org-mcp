#+title: org-mcp
#+author: Kai Lee

An [[https://modelcontextprotocol.io][MCP]] (Model Context Protocol) server that exposes Org-mode's data model over stdio JSON-RPC 2.0.

* Requirements

- Emacs 29.1+
- Org-mode 9.6+

* Installation

With =use-package= (Emacs 29+):

#+begin_src emacs-lisp
(use-package org-mcp
  :vc (:url "https://github.com/hsienchiaolee/org-mcp"))
#+end_src

With =use-package= + =straight.el=:

#+begin_src emacs-lisp
(use-package org-mcp
  :straight (org-mcp :type git :host github :repo "hsienchiaolee/org-mcp"))
#+end_src

With =elpaca=:

#+begin_src emacs-lisp
(elpaca (org-mcp :host github :repo "hsienchiaolee/org-mcp"))
#+end_src

With Doom Emacs (in =packages.el=):

#+begin_src emacs-lisp
(package! org-mcp :recipe (:host github :repo "hsienchiaolee/org-mcp"))
#+end_src

Manual:

#+begin_src sh
git clone https://github.com/hsienchiaolee/org-mcp.git ~/.emacs.d/site-lisp/org-mcp
#+end_src

#+begin_src emacs-lisp
(add-to-list 'load-path "~/.emacs.d/site-lisp/org-mcp")
(require 'org-mcp)
#+end_src

* Usage

** Batch mode

Run as a standalone process:

#+begin_src sh
emacs --batch -Q -L /path/to/org-mcp -l org-mcp -f org-mcp-start
#+end_src

** Daemon mode

Start from a running Emacs daemon:

#+begin_src emacs-lisp
(org-mcp-start)
#+end_src

** MCP client configuration

Add to your MCP client config (e.g. =claude_desktop_config.json=):

#+begin_src json
{
  "mcpServers": {
    "org-mcp": {
      "command": "emacs",
      "args": ["--batch", "-Q", "-L", "/path/to/org-mcp", "-l", "org-mcp", "-f", "org-mcp-start"]
    }
  }
}
#+end_src

* Access Control

By default, org-mcp restricts file access to directories containing your =org-agenda-files=. To explicitly configure which directories are accessible:

#+begin_src emacs-lisp
(setq org-mcp-allowed-directories '("~/org/" "~/projects/plans/"))
#+end_src

Any tool call that attempts to read or write a file outside these directories will be rejected with a JSON-RPC error. When =org-mcp-allowed-directories= is =nil= (the default), the directories containing your =org-agenda-files= are used.

* Audit Logging

To log mutation tool calls (=org_set_state=, =org_set_property=, =org_set_headline=, =org_append_body=, =org_capture=) to a file:

#+begin_src emacs-lisp
(setq org-mcp-log-file "~/org/mcp-audit.log")
#+end_src

When set, each mutation is appended as a timestamped line. Disabled by default.

* Tools

| Tool                 | Description                                        |
|----------------------+----------------------------------------------------|
| =org_get_entry=      | Return full data for an entry by org-id            |
| =org_query=          | Query agenda files with s-expression filter        |
| =org_get_children=   | Return immediate children of an entry              |
| =org_get_properties= | Return the property drawer (with optional inherit) |
| =org_set_state=      | Set or clear the TODO keyword                      |
| =org_set_property=   | Set or delete a property                           |
| =org_append_body=    | Append text to body, optionally in a drawer        |
| =org_capture=        | Create a new entry from inline params or template  |

* Notifications

When enabled, the server emits MCP notifications for Org-mode events:

- =org/entryStateChanged= — TODO state transitions
- =org/propertyChanged= — property modifications
- =org/entryCreated= — new entries created via =org_capture=
